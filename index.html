SAP Integration Content Automation

This document provides a comprehensive guide for using the Integration Content - Automation Postman Collection, designed to automate interaction with the SAP Integration Suite OData API for Integration Content.

1. üöÄ Overview and Prerequisites

Collection Purpose

  • Content Discovery & Backup (Folder 01/1.1): Retrieve a list of all Integration Packages and Flows (iFlows), their parameters, and optionally download them as .zip files.
  • Mass Configuration Update (Folder 02): Update configuration parameters for multiple iFlows based on a provided CSV data file.
  • Mass Deployment/Undeployment (Folders 03, 04, 05, 06): Deploy or undeploy various artifact types (iFlows, Script Collections, Value Mappings) using a data file.

Mandatory Prerequisites

Before running any request, you must complete the following setup steps in your SAP BTP account and Postman:

Step 1: Create Service Key with Required Roles

You must create a Service Instance for Process Integration Runtime (Plan: api) in the SAP BTP Cockpit. Ensure the service key is configured with the following minimum roles to allow full functionality of this collection:

  • WorkspacePackagesTransport
  • WorkspacePackagesEdit
  • WorkspacePackagesRead
  • WorkspacePackagesConfigure
  • WorkspaceArtifactsDeploy

Note: The ClientID, ClientSecret, and TokenUrl required below can be found in the JSON output of this Service Key.

Step 2: Configure Collection Variables

Configure the following variables in the Collection Variables tab within Postman:

Variable Description
baseUrl The base URL for the Integration Content OData API (found in Service Key as url appened by /api/v1).
TokenUrl The URL for fetching the OAuth token (found in Service Key as tokenurl).
ClientID The OAuth Client ID (found in Service Key as clientid).
ClientSecret The OAuth Client Secret (found in Service Key as clientsecret).

Step 3: Setup Local Node.js Server

This collection requires a local Node.js server to save downloaded files (e.g., iFlow .zip backups) and CSV reports generated during execution. Follow these detailed steps:

A. Install Node.js
  1. Go to the official Node.js Website.
  2. Download the LTS (Long Term Support) version for your operating system.
  3. Run the installer.
    • Important: During installation, ensure the option "Add to PATH" (or "Add to environment variables") is selected. This allows you to run node commands from any folder.
B. Verify Installation & Environment Variables

To ensure Node.js is installed and the PATH is set correctly:

  1. Open a new Command Prompt (Windows) or Terminal (Mac/Linux).
  2. Type node -v and press Enter. You should see a version number (e.g., v18.16.0).
  3. Type npm -v and press Enter. You should see a version number for the package manager.

Troubleshooting: If you get an error saying "node is not recognized", you may need to manually add the Node.js installation path (usually C:\Program Files\nodejs\ on Windows) to your System Environment Variables under "Path".

C. Setup and Run the Server
  1. Download the saveFilesServer.js file from this GitHub repository.
  2. Create a new folder on your computer (e.g., postman-server) and move the saveFilesServer.js file into it.
  3. Open a terminal inside that folder.
  4. Run the following commands to initialize the project and install dependencies:
    npm init -y
    npm install express
  5. Start the server by running:
    node saveFilesServer.js
  6. You should see a message: üöÄ Server is running and listening at http://localhost:3000. Keep this terminal window open while running Postman.

3. üõ†Ô∏è Detailed Workflows

3.1. Folder 01: Get Integration Content (Full Scan)

This workflow scans all integration packages in your tenant to gather lists, configurations, and optionally download the files.

How to Run

  1. Set the control variables (runGetConfigurations, runDownloadFlow, nodeServerUrl) in the Collection Variables tab based on your requirements.
  2. Select the Folder to Run: In the Postman sidebar, click the three dots (...) next to the folder 01 - Get Integration Content and select Run.
  3. In the Runner window, expand Advanced Settings.
  4. Check "Run collection without using stored cookies".
  5. Uncheck "Save cookies after collection run".
  6. Run the Collection: Click Run Integration Content - Automation.

Workflow Steps

Request Name Method Purpose & Functionality
Get Auth Token POST Fetches the accessToken. Clears old tokens and optionally clears cookies.
Get all integration packages GET Step 1: Discover Packages. Retrieves a list of all packages. Initializes loop variables and summary variables.
Get all integration flows of a package. GET Step 2: Loop through Packages. For the current package, it retrieves all contained iFlows. Updates sets up the inner iFlow loop.
Get configurations of an integration flow... GET Step 3a: Get Configurations. (If runGetConfigurations=true). Retrieves config parameters. Appends data to csvData.
Download an integration flow... GET Step 3b: Download Files. (If runDownloadFlow=true). Downloads the binary. Sends to local server. Acts as loop controller.
Output Results

After the run completes, check the following Collection Variables for the results:

Variable Name Description
TotalPackageCount Count of all Integration Packages found.
TotalIflowCount Count of all Integration Flows found.
PackageCsvList A CSV list of all Integration Packages found.
IflowCsvList A CSV list of all Integration Flows found (Package Name, iFlow Name).
parametersCsvOutput The final CSV containing all configuration parameters (if runGetConfigurations=true).

3.2. Folder 1.1: Get Specific Integration Content (Specific Scan)

This is an optimized workflow designed to fetch configurations for only a specific list of packages defined in the specificPackageIDsList variable.

How to Run

  1. Set the specificPackageIDsList variable with a comma-separated list of the Package IDs you want to process (e.g., ID_A,ID_B,ID_C).
  2. Select the Folder to Run: In the Postman sidebar, click the three dots (...) next to the folder 1.1 - Get Specific Integration Content and select Run.
  3. In the Runner window, expand Advanced Settings.
  4. Check "Run collection without using stored cookies".
  5. Uncheck "Save cookies after collection run".
  6. Run the Collection: Click Run Integration Content - Automation.

Workflow Steps

Request Name Method Purpose & Functionality
Logic: Specific Package Iteration GET Main router: initializes the list on iteration 0 and loops through packages on subsequent calls.
Get Auth Token (Specific) POST Fetches the required access token.
Get Package Details by ID GET Retrieves the Package Name for reporting purposes.
Get all integration flows... GET Retrieves the iFlows for the currently processed package.
Get configurations... GET Retrieves configurations. Appends to specificCsvData. Loops to next iFlow/Package.
Output Results

After the run completes, check the following Collection Variable for the results:

Variable Name Description
specificParametersCsvOutput The final CSV containing configuration parameters for the specified packages only.

3.3. Folder 02: Update Iflow Configuration (Mass Update)

This workflow is used to mass update configuration parameters of iFlows using the Postman Collection Runner and an attached data file.

Data File Preparation (CSV Headers)

You must create a data file (CSV) with the following mandatory headers. Each row will represent one configuration parameter to be updated:

Header Description Example Value
PackageName The Name of the Package Containing Iflow (Optional). ERP_Sync
PackageID The ID of the Package Containing Iflow (Optional). PKG_ERP_SYNC
IflowName The Name of the Integration Flow artifact (Optional). ERP_To_S4
IflowID The ID of the Integration Flow artifact. IFLOW_S4
Version The version of the Integration Flow (e.g., specific version or active). 1.0.0 or active
ParameterKey The technical key of the configuration parameter (e.g., Receiver, Endpoint URL). Receiver_SOAP_Host
ParameterValue The new value for the parameter. https://new.endpoint.com/service
DataType The expected data type of the parameter (usually xsd:string). xsd:string

Example of Prepared CSV Data

Below is a visual example of how your CSV file should look:

PackageName PackageID IflowName IflowID Version ParameterKey ParameterValue DataType
ERP_Sync PKG_ERP_SYNC ERP_To_S4 IFLOW_S4 active Receiver_Host https://s4hana.new.com/api xsd:string
CRM_Sync PKG_CRM_SYNC CRM_To_SF IFLOW_SF active Client_ID newClientIDvalue xsd:string
HCM_Flows PKG_HCM Hiring_SF IFLOW_HR 1.0.5 Sender_URL_Path /new/path/hcm xsd:string

Raw CSV Input (Copy/Paste):

PackageName,PackageID,IflowName,IflowID,Version,ParameterKey,ParameterValue,DataType
ERP_Sync,PKG_ERP_SYNC,ERP_To_S4,IFLOW_S4,active,Receiver_Host,https://s4hana.new.com/api,xsd:string
CRM_Sync,PKG_CRM_SYNC,CRM_To_SF,IFLOW_SF,active,Client_ID,newClientIDvalue,xsd:string
HCM_Flows,PKG_HCM,Hiring_SF,IFLOW_HR,1.0.5,Sender_URL_Path,/new/path/hcm,xsd:string

How to Run

  1. Prepare the Data File: Create the CSV file as specified above.
  2. Select the Folder to Run: In the Postman sidebar, click the three dots (...) next to the folder 02 - Update Iflow Configuration and select Run.
  3. Load the Data File: Click Select File and upload your prepared CSV file.
  4. In the Runner window, expand Advanced Settings.
  5. Check "Run collection without using stored cookies".
  6. Uncheck "Save cookies after collection run".
  7. Run the Collection: Click Run Integration Content - Automation.

Workflow Steps (Runner Logic)

Request Name Method Iteration 0 (Setup) Iteration ≥ 1 (Loop) Purpose
Iteration Check GET Clears tokens, initializes csvResults, proceeds to Get Auth Token. Jumps directly to Update configuration... The router that controls efficient looping.
Get Auth Token POST Fetches accessToken, proceeds to Get CSRF token. Skipped Authorizes the session (once per run).
Get CSRF token GET Fetches csrfToken, proceeds to Update configuration.... Skipped Fetches the token required for the PUT (write) operation (once per run).
Update configuration parameter... PUT Executes the update for the first data row. Executes the update for the current data row. Updates the config and appends to csvResults.
Output Results

After the run completes, check the following Collection Variable for the results:

Variable Name Description
csvResults A CSV string containing the input data plus the ResponseCode and ResponseMessage for each update attempt.
